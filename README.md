| 방법 | 학습된 w (정답: 2.0) | 학습된 b (정답: -1.0) | x=10 예측 (정답: 19.0) |

## 실습 질문 2: 데이터를 바꿔서 다른 공식을 학습시켜 보세요. 세 방법 모두 잘 맞추나요?

원본 예시(`y = 3x + 2`)를 피하고 **`y = 4x - 7`** 로 실험했습니다.

- X: [-1, 0, 1, 2, 3, 4]
- y (정답): [-11, -7, -3, 1, 5, 9]
- 노이즈: scale=1.0 동일 적용

| 방법 | 학습된 w (정답: 4.0) | 학습된 b (정답: -7.0) | x=10 예측 (정답: 33.0) |

| 신경망 | 3.8995 | -6.4851 | 32.51 |
| NumPy Polyfit | 3.9124 | -6.5251 | 32.60 |
| SciPy CurveFit | 3.9124 | -6.5251 | 32.60 |

**결론**: 세 방법 모두 잘 맞춥니다. 공식이 바뀌어도 각 방법이 올바르게 기울기와 절편을 찾아냅니다. 노이즈로 인해 완벽히 일치하지는 않지만, 모두 정답에 근접한 값을 출력했습니다.

## 실습 질문 3: 노이즈 크기(scale=1.0)를 변경하면 각 방법의 결과가 어떻게 달라지나요?

`y = 4x - 7` 데이터에서 노이즈 scale만 바꿔 NumPy Polyfit으로 실험했습니다.

| 노이즈 scale | 학습된 w (정답: 4.0) | 학습된 b (정답: -7.0) | 오차 |
|-------------|---------------------|----------------------|------|
| 0.1 (노이즈 거의 없음) | **3.9912** | **-6.9525** | 매우 작음 |
| 1.0 (기본값) | 3.9124 | -6.5251 | 보통 |
| 5.0 (노이즈 매우 큼) | 3.5619 | -4.6255 | 큼 |

**결론**: 노이즈가 클수록 세 방법 모두 정답에서 멀어집니다. 특히 `scale=5.0`에서는 b 값이 -4.6으로 정답 -7.0에서 크게 벗어났습니다. 반면 `scale=0.1`에서는 거의 완벽한 값(w=3.99, b=-6.95)을 찾아냈습니다.

이를 통해 **데이터의 품질(노이즈 수준)이 모든 학습 방법의 정확도에 직접적인 영향을 미친다**는 것을 확인할 수 있었습니다.

## 추가 관찰: y = 2x - 1 vs y = 4x - 7, Training Loss 모양이 왜 다른가?

두 신경망의 학습 손실 그래프를 비교했을 때 눈에 띄는 차이가 있었습니다.

- `y = 2x - 1`: 초기 Loss ≈ 12.5, 약 50 epoch 안에 빠르게 수렴
- `y = 4x - 7`: 초기 Loss ≈ 55, 200 epoch 이후까지 완만하게 감소

**원인 분석**

**1. 초기 Loss 스케일 차이**
Loss(MSE) = (예측 - 정답)²의 평균입니다. y 값의 범위가 클수록 초기 오차도 제곱으로 커집니다.
- `y = 2x - 1`의 y 범위: -3 ~ 7
- `y = 4x - 7`의 y 범위: -11 ~ 9 → 초기 Loss가 약 4배 이상 큼

**2. 같은 보폭으로 더 먼 거리를 걸어야 함**
SGD의 learning rate는 기본값 0.01로 고정되어 있습니다. 목적지(정답 파라미터)까지의 거리가 다릅니다.
- `y = 2x - 1`: w: 0 → 2, b: 0 → -1
- `y = 4x - 7`: w: 0 → 4, b: 0 → **-7** ← 특히 b가 멀다

같은 보폭인데 목적지가 훨씬 멀기 때문에 더 많은 epoch이 필요합니다.

**3. 그래프 모양 착시**
y축 스케일 자체가 달라서(12 vs 55) 시각적으로 `4x-7`이 더 완만해 보이는 효과도 있습니다. 실제 수렴 속도(비율)는 비슷합니다.

**시사점**: 데이터의 y 값 스케일이 클수록 신경망 학습이 느려질 수 있습니다. 실제 딥러닝에서 데이터를 정규화(normalization)하는 이유가 바로 이것입니다. 데이터를 0 ~ 1 또는 -1 ~ 1 범위로 맞춰주면 Loss 스케일이 균일해져 훨씬 빠르게 수렴합니다.
